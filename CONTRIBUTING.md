# Contributing

Thanks for contributing to EducativeViewer.

## Before You Start

- Read [README.md](README.md) for local setup.
- Read [Cloudflare_Vercel.md](Cloudflare_Vercel.md) for proxy and routing behavior.
- For local frontend run, use deploy script flow (option 2 or option 4).

## Development Workflow

1. Create a branch from your working branch.
2. Make focused changes.
3. Update docs when behavior changes.
4. Open a pull request with a clear summary.

## Commit Guidelines

- Keep commits small and meaningful.
- Use clear commit messages (for example: fix proxy route handling).
- Avoid bundling unrelated changes in one commit.

## Pull Request Checklist

- Code builds/runs locally.
- Routing behavior is verified.
- README/docs are updated if needed.
- No sensitive values are committed.

## Need Help

Open an issue with:
- what you changed,
- expected behavior,
- actual behavior,
- logs or screenshots.
